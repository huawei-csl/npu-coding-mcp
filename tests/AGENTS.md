# AGENTS.md

## Setup

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

# Build the AscendC FTS5 search index (one-time, index is ~11 MB and gitignored)
python -m npu_coding_mcp build-ascendc-index
```

CI uses `uv sync --all-groups` and `uv run` — works as an alternative to pip.

## Commands

```bash
# Lint + format check
ruff check src/ tests/
ruff format --check src/ tests/

# Fix lint + format
ruff check --fix src/ tests/
ruff format src/ tests/

# Run tests (requires pto-isa docs)
PTOISA_DOCS_PATH=/path/to/pto-isa/docs pytest tests/ -v

# Run server (stdio mode for MCP clients)
python -m npu_coding_mcp serve --stdio

# Run server with explicit pto-isa docs path (HTTP mode, default 0.0.0.0:8080)
python -m npu_coding_mcp serve /path/to/pto-isa/docs

# Rebuild AscendC index
python -m npu_coding_mcp build-ascendc-index --ascendc-docs data/ascendc_docs --ascendc-index data/ascendc_index.db
```

The CLI accepts bare positional args without the `serve` subcommand for backward compat: `python -m npu_coding_mcp /path/to/docs --stdio`.

## Gitignored data files

These are not committed but needed at runtime or for full test coverage:

| File | Source | Purpose |
|------|--------|---------|
| `data/ascendc_index.db` | `build-ascendc-index` | AscendC FTS5 search index |
| `data/pto_isa_headers/` | Snapshot during CCE mapping build | NPU headers for CCE extraction |

The server auto-builds `ascendc_index.db` from `data/ascendc_docs/` on startup if missing.

## Testing

Tests require access to the real `pto-isa` documentation tree. Set `PTOISA_DOCS_PATH` env var or place the `pto-isa` repo as a sibling directory (`../pto-isa/docs`). Without docs, all tests fail.

CCE mapping tests (`test_cce_extractor.py`) additionally require `data/cce_mapping.json` (committed) to exist.

## Architecture

- **src layout**: single package at `src/npu_coding_mcp/`
- **Two domains**: PTO-ISA (14 tools in `tools.py`) + AscendC (6 tools in `ascendc/tools.py`) = 20 total
- **Startup**: `loader.py` parses all markdown docs into an in-memory `ISAStore` (Pydantic models in `models.py`). The AscendC loader lives in `ascendc/loader.py`
- **Auto-clone**: when no `docs_path` given, `repo_manager.py` clones `https://gitcode.com/cann/pto-isa.git` into `~/.cache/npu-coding-mcp/`
- **Transport**: FastMCP supports both `--stdio` and HTTP (`streamable-http`)
- **CCE mapping**: Pre-built via compile-and-extract in `data/cce_mapping.json`. `loader.py` loads it into `ISAStore.cce_mappings`. Regenerate with `scripts/build_cce_mapping.py`
- **Entry point**: `__main__.py` — subcommands `serve` (default) and `build-ascendc-index`

## CCE Mapping (`get_cce_intrinsics` tool)

Maps each PTO-ISA instruction to the CCE (CANN Compute Engine) intrinsics it calls in NPU backend headers. The mapping is in `data/cce_mapping.json` (committed, tracks pto-isa commit in `data/ptoisacommit.txt`) and generated by `scripts/build_cce_mapping.py` via **compile-and-extract**: each NPU header is compiled with `g++ -E` to resolve all templates and `if constexpr` branches, then CCE intrinsic calls are regex-extracted from the preprocessed output. No LLM needed for extraction.

```bash
# Rebuild cce_mapping.json (fast, ~100s)
python scripts/build_cce_mapping.py --force --no-llm

# Rebuild with LLM context enrichment (adds human-readable descriptions)
python scripts/build_cce_mapping.py --force --api-key sk-xxx --delay 1
```

LLM is optional — only used for `context` field enrichment when `--api-key` is provided.

Requires: `g++` on PATH, PTO headers at `/usr/local/Ascend/cann-9.0.0/x86_64-linux/include`. Set `GXX` env var to override compiler path.

## AscendC documentation

- 1,771 markdown sections across 7 chapters, all in Chinese, pre-extracted to `data/ascendc_docs/` (committed)
- FTS5 index is built at `data/ascendc_index.db` (gitignored, auto-built on startup if missing)
- All 6 AscendC tools accept a `language` param: `"en"` (default, prepends translation instruction) or `"zh"` (raw Chinese)
- Tools: `search_docs`, `get_section`, `list_chapters`, `get_chapter_tree`, `search_api`, `get_toc`

## Known quirks

- `pyproject.toml` has `packages = ["src/pto_isa_mcp"]` under `[tool.hatch.build.targets.wheel]` — this is stale; the real package is `src/npu_coding_mcp`. Editable installs work fine (`pip install -e`), but wheel builds may need this fixed.
- The manifest file is named `manifest.yaml` but is actually JSON. The loader handles both.
