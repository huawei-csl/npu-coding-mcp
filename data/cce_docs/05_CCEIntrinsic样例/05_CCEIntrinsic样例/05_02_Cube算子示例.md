# Cube 算子示例

> **Section**: 5.2


本节将介绍 2 例 Cube 算子的用法，其主要区别在于数据搬入的路径不同。

具体来说，示例 1 的路径为将矩阵从 GM 直接搬入 L0A 和 L0B ，之后经过矩阵相乘得到存 于 L0C 的结果，再将 L0C 的结果搬出至 GM 。在该数据搬入的路径中，由 GM 到 L0A 和 L0B 的数据搬运接口本身不具备分形转换的能力，需要保证在 GM 中的数据已经是 ZZ （ L0A ）和 ZN （ L0B ）格式的矩阵数据。

- lenBurst ：一次连续搬运数据长度，单位 32Bytes ，元素个数为 64 ， 64 * sizeof(float) = 256B ，计算搬运长度 256/32=8 。
- srcGap ：两次搬运源地址间隙，源地址处连续搬运，此处设置 0 。
- dstGap ：两次搬运目的地址间隙，目标地址处连续搬运，此处设置 0 。
- byteMode ：通过设置 byteMode 枚举参数可改变 block 大小。设为 bm\_t::BM\_DISABLE ，即 block 大小为 32Bytes 。
