# 接口基本信息

> **Section**: 6.5.6.1


将 L0\_SET\_VALUE 寄存器中的 16 位 /32 位数据广播到 dst 中的多个块中。

## 所属流水线：

- 对于 L0A 和 L0B 为目的地址，该接口在 MTE1 流水线中执行。

## 接口原型

- 对于 L1 为目的地址，该接口在 MTE2 流水线中执行。

## 数据对齐要求：

- 对于 L0A/L0B ，地址应为 512 字节对齐，每个数据块为 512 字节。
- 对于 L1 ，地址应为 32 字节对齐，每个数据块为 32 字节。

接口中用到的 config 是一个 64 位参数，不同位数设置不同内容，具体内容如下表：

表 6-23 config 位数说明

| 位数             | 说明                                                                                                       | 取值范围        | 单位                                       |
|----------------|----------------------------------------------------------------------------------------------------------|-------------|------------------------------------------|
| config[ 14:0]  | 重复次数，每次重复时， src 中 的地址将增加 (config[46:32] + config[30:16]) * block_size ； 如果希望执行一次接口，则设 置 config[14:0]=1 。 | [0, 2^15-1] | /                                        |
| config[ 30:16] | 每次重复的块数量 （ block_size ）。                                                                                 | [0, 2^15-1] | /                                        |
| config[ 46:32] | 两次连续重复之间的间隔 （ Gap ）。 第一次搬运的尾地址与第二次 搬运的首地址的间隔。                                                            | [0, 2^15-1] | 对于 L0A/L0B ，单 位为 512 字节 对于 L1 ，单位为 32 字节 |
